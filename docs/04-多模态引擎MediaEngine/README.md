# MediaEngine — 多模态内容分析 Agent

## 文件位置
- `MediaEngine/agent.py` (20KB) — Agent 主逻辑
- `MediaEngine/llms/` — LLM 接口封装
- `MediaEngine/nodes/` — 处理节点
- `MediaEngine/tools/` — 多模态搜索工具集
- `MediaEngine/state/` — 状态管理
- `MediaEngine/prompts/` — 提示词模板
- `MediaEngine/utils/` — 工具函数

## 功能定位

**多模态理解 Agent**，负责：
- 解析抖音、快手等短视频内容
- 提取图片中的文字和信息
- 解析现代搜索引擎中的结构化多模态信息卡片（天气、日历、股票等）
- 理解图文混合内容

## 推荐模型
- **Gemini 2.5 Pro**（官方推荐，具备最强多模态能力）
- 推荐中转厂商：https://aihubmix.com/

## 核心能力

### 1. 视频内容深度解析
- 下载短视频并提取关键帧
- 语音转文字（ASR）
- 画面内容理解（OCR + 场景识别）
- 视频情感基调判断

### 2. 图片分析
- 文字识别（OCR）
- 物体/场景识别
- 信息卡片结构化提取
- 图表数据提取

### 3. 多模态搜索
- 以图搜图
- 视频关键帧检索
- 跨模态内容匹配

## 工作流程

```
用户问题
    │
    ▼
┌──────────────────┐
│  内容获取         │ ← 爬取多模态内容（视频/图片/图文）
└────────┬─────────┘
         ▼
┌──────────────────┐
│  多模态解析       │ ← LLM 视觉能力分析
│  (图片理解+视频)   │
└────────┬─────────┘
         ▼
┌──────────────────┐
│  信息结构化       │ ← 提取关键信息，格式化输出
└────────┬─────────┘
         ▼
┌──────────────────┐
│  反思与补充       │ ← 发现缺失信息→补充分析
└────────┬─────────┘
         ▼
┌──────────────────┐
│  最终输出         │ → 多模态分析结果
└──────────────────┘
```

## 结构特点

与 QueryEngine 高度一致的模块化结构：

```
MediaEngine/
├── agent.py      # Agent主类，整合流程
├── llms/         # LLM客户端（OpenAI兼容接口）
├── nodes/        # 处理节点（搜索→总结→反思→再总结）
├── tools/        # 多模态搜索工具（搜索引擎+视频下载等）
├── state/        # Agent状态管理
├── prompts/      # 提示词模板
└── utils/        # 辅助函数
```

## 在系统中的角色

在"论坛"协作机制中，MediaAgent 提供 **多模态内容视角**：
- 识别纯文本无法获取的视觉信息
- 分析短视频平台的情绪表达
- 理解信息图表和数据可视化内容
